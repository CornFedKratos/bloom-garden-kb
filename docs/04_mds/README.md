## What This Is
The Bloom Garden Knowledge Base is the organizational memory of the project. It is the source of truth for decisions, clinical evidence, compliance architecture, and product rationale. All agents, OT advisors, and collaborators read relevant KB sections before contributing.

Markdown files in this KB are the source of truth. Exports are renders.

## Decision Record Format
All significant decisions use this format in `/02-product-decisions/`:

```markdown
# Decision Record — [Name]
**Date:**
**Status:** Proposed / Validated / Approved / Rejected
**Decision Maker:** Don + [OT Advisor if applicable]

## What Was Decided
## Why
## Alternatives Considered
## Clinical Evidence
## CIL Status
## Open Questions
## Review Trigger
```

---

## KB Maintenance Rules
1. Every significant product decision gets a decision record
2. Every clinical mechanic gets a Clinical Evidence Brief before it enters spec
3. Every rejected feature gets a record in `/02-product-decisions/rejected-features/`
4. Open questions are logged — never left in conversation only
5. This KB is the first thing a new agent or collaborator reads
